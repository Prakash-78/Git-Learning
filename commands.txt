# Commands to initialize and push the files to repository
# Date 17/02/26
-> git init / git clone ("clone the repository from github)
-> git add . / git add <filename>   (First one let'syou to commit all the unstaged/newly created files & second one let's you to add certain file)
-> git commit -m "Message" (Committing/moving the completed files to staging)
->git status (It will let you see the unstaged / new file created in the files )
->git branch -M main (Helps you to rename your branch name (-M -> forces move/rename and (min -> file name)))
-> git remote add origin 'https://github.com/Prakash-78/Git-Learning.git' ("It helps the git to track in which repository we have to add and commit the files)
-> git push -u origin main(→ Sets up tracking between local main and remote main.)


# Branching helps you to work on new features and fixing bugs and errors without disturbing the main code 
->git branch <branchname>   (Creating a new branch)   /( git checkout -b <name>) -> creates a new branch
->git branch -b <branchname>
->git switch -c feature3 (Creates branch and switches to it helps reducing the creationg and switching from main)
-> git switch <branchname> (helps switching between branches)
-> git branch -d <branchname> (deletes a branch)
-> git branch -D <branchname> (force deletes a branch)

# Diffferent staged/unstaged changes
-> git diff HEAD    (difference all the staged and unstaged changes)
->git diff --staged   (diff just staged changes)
-> git diff     (Diff just unstaged changes)


# Different between commit
-> git show <commit>    ("Helps in identifying what are all the updates made to the files)
->git diff <commit1> <commit2>    (Diff two commits)
->git diff <commit> <file>  (Diff one file since a commit)
->git diff <commit> --stat / ->git show <commit> --stat     (Show a summary of a diff)
